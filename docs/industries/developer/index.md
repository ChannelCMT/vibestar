# 前端 / 后台开发的 Vibe Coding 工作流

<section class="industry-hero">
  <p>Industry Scenario 03</p>
  <h2>从写代码的人，升级为管理 AI 开发流水线的人。</h2>
  <span>开发 / 调试 / 接口联调 / 代码审查 / 自动化脚本</span>
</section>

开发者使用 AI，不是为了放弃工程判断，而是把重复编码、文档查找、样板代码、测试补齐和问题定位交给 AI 加速。真正的价值在于：你能定义架构边界、审查 AI 输出，并用 Agent 工作流把开发、测试、文档和评审串起来。

## 典型任务

<section class="industry-tasks">
  <article><h3>功能开发</h3><p>根据需求生成组件、接口、数据结构和业务逻辑。</p></article>
  <article><h3>调试联调</h3><p>定位报错、解释日志、补充断言、生成接口 mock 和排查步骤。</p></article>
  <article><h3>工程质量</h3><p>补测试、做重构、生成文档、检查边界条件和安全风险。</p></article>
</section>

## 三种工作流对比

<section class="workflow-grid">
  <article><strong>01</strong><h3>古法工作流程</h3><p>手写功能、查文档、复制样板、定位报错，再手动补测试和说明。</p><ul><li>重复代码消耗时间</li><li>上下文切换频繁</li><li>测试和文档容易滞后</li><li>代码评审依赖人工经验</li></ul></article>
  <article><strong>02</strong><h3>对话式 AI 工作流程</h3><p>把 AI 当成结对开发者，让它解释代码、生成实现、补测试和提出风险。</p><ul><li>生成组件和接口代码</li><li>解释复杂函数和历史逻辑</li><li>根据报错生成修复方案</li><li>自动生成 README 和测试用例</li></ul></article>
  <article><strong>03</strong><h3>Agents 工作流</h3><p>把开发任务拆给多个 Agent：实现、测试、文档、审查各自负责。</p><ul><li>开发 Agent 完成功能代码</li><li>测试 Agent 补单测和边界用例</li><li>Reviewer Agent 检查质量风险</li><li>Docs Agent 更新说明和示例</li></ul></article>
</section>

## 一个开发者的实操闭环

<section class="industry-process">
  <div><span>Step 1</span><h3>输入需求</h3><p>明确功能目标、接口约束、技术栈和验收标准。</p></div>
  <div><span>Step 2</span><h3>生成实现</h3><p>让 AI 创建代码骨架、核心逻辑和必要的配置文件。</p></div>
  <div><span>Step 3</span><h3>运行测试</h3><p>用 AI 补测试、解释失败原因，并持续修复直到跑通。</p></div>
  <div><span>Step 4</span><h3>评审交付</h3><p>让 Agent 检查可维护性、安全性、性能和文档完整度。</p></div>
</section>

## 课堂练习

你将选择一个真实开发任务，例如“用户登录页”“数据上传接口”“自动报表脚本”，并完成：

- 一个可运行功能模块
- 一组测试用例
- 一份代码审查记录
- 一份接口或使用文档
- 一次 Agent 协作修复流程
