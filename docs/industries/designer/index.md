# 设计师的 Vibe Coding 工作流

<section class="industry-hero">
  <p>Industry Scenario 04</p>
  <h2>从视觉方案，到可体验的交互原型。</h2>
  <span>视觉概念 / 交互原型 / 品牌页面 / 演示素材</span>
</section>

设计师的 AI 工作流不只是生成图片，而是把风格、文案、交互、组件和页面 Demo 连接起来。Vibe Coding 能帮助设计师用自然语言快速搭建可体验原型，让沟通从“看静态稿”变成“直接试一试”。

## 典型任务

<section class="industry-tasks">
  <article><h3>风格探索</h3><p>生成情绪板、配色方向、品牌语气和视觉关键词。</p></article>
  <article><h3>交互原型</h3><p>把流程、状态、按钮和反馈机制变成可点击页面。</p></article>
  <article><h3>交付表达</h3><p>输出组件说明、设计理由、展示页面和客户汇报材料。</p></article>
</section>

## 三种工作流对比

<section class="workflow-grid">
  <article><strong>01</strong><h3>古法工作流程</h3><p>先做静态稿，再反复导图、写说明、等开发还原。</p><ul><li>交互效果难表达</li><li>静态稿无法验证体验</li><li>设计和开发沟通成本高</li><li>演示材料需要重复制作</li></ul></article>
  <article><strong>02</strong><h3>对话式 AI 工作流程</h3><p>让 AI 生成页面结构、文案、交互状态和代码原型。</p><ul><li>快速生成多版风格方向</li><li>补全界面文案和空状态</li><li>把设计稿转成 HTML 原型</li><li>生成客户汇报话术</li></ul></article>
  <article><strong>03</strong><h3>Agents 工作流</h3><p>让品牌、交互、前端和评审 Agent 协作形成可体验 Demo。</p><ul><li>品牌 Agent 定义视觉基调</li><li>交互 Agent 设计流程状态</li><li>原型 Agent 生成页面代码</li><li>评审 Agent 检查一致性</li></ul></article>
</section>

## 一个设计师的实操闭环

<section class="industry-process">
  <div><span>Step 1</span><h3>定义风格</h3><p>输入品牌定位、目标用户、场景和参考方向。</p></div>
  <div><span>Step 2</span><h3>生成结构</h3><p>让 AI 输出页面信息架构、组件清单和文案草稿。</p></div>
  <div><span>Step 3</span><h3>制作原型</h3><p>用 Vibe Coding 生成可点击页面，验证布局和交互。</p></div>
  <div><span>Step 4</span><h3>评审优化</h3><p>让 Agent 从品牌一致性、可用性和开发可行性角度提建议。</p></div>
</section>

## 课堂练习

你将选择一个品牌或产品主题，例如“AI 课程首页”“报名转化页”“活动宣传页”，并完成：

- 一份视觉风格 brief
- 一套页面文案和组件说明
- 一个可预览 HTML 原型
- 一份设计评审记录
- 一份面向客户或老板的展示稿
