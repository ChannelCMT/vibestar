# 产品经理的 Vibe Coding 工作流

<section class="industry-hero">
  <p>Industry Scenario 01</p>
  <h2>从写文档的人，升级为能驱动产品原型和 Agent 系统的人。</h2>
  <span>古法工作流程 / 对话式 AI 工作流程 / Agents 工作流</span>
</section>

产品经理的核心工作不是“写更多文档”，而是把用户问题、业务目标、技术约束和团队协作组织成可以落地的产品方案。Vibe Coding 让产品经理从“描述需求的人”，进一步变成“能用 AI 生成原型、验证方案、编排执行”的产品创造者。

## 典型任务

<section class="industry-tasks">
  <article>
    <h3>需求洞察</h3>
    <p>整理用户反馈、访谈纪要、竞品信息和业务数据，提炼真正值得解决的问题。</p>
  </article>
  <article>
    <h3>方案设计</h3>
    <p>输出用户故事、功能范围、信息架构、交互流程、PRD 和验收标准。</p>
  </article>
  <article>
    <h3>原型验证</h3>
    <p>快速生成页面原型、Demo、测试脚本和汇报材料，用真实反馈迭代方案。</p>
  </article>
</section>

## 三种工作流对比

<section class="workflow-grid">
  <article>
    <strong>01</strong>
    <h3>古法工作流程</h3>
    <p>依靠人工收集资料、手写文档、开会对齐，再等待设计和开发排期。</p>
    <ul>
      <li>大量时间花在资料整理和文档排版上</li>
      <li>需求表达依赖个人经验，颗粒度不稳定</li>
      <li>原型验证慢，很多想法停留在 PPT 或 PRD</li>
      <li>跨团队沟通成本高，反复解释背景和细节</li>
    </ul>
  </article>
  <article>
    <strong>02</strong>
    <h3>对话式 AI 工作流程</h3>
    <p>把 AI 当作产品助理，通过对话完成拆解、总结、生成和优化。</p>
    <ul>
      <li>让 AI 总结用户反馈，提炼问题和机会点</li>
      <li>让 AI 生成用户故事、PRD 大纲和验收标准</li>
      <li>让 AI 扮演用户、开发、设计和老板做方案评审</li>
      <li>用 AI 快速生成汇报文案、竞品表格和测试问题</li>
    </ul>
  </article>
  <article>
    <strong>03</strong>
    <h3>Agents 工作流</h3>
    <p>把高频产品任务拆给多个 Agent，让它们协作完成从研究到原型的闭环。</p>
    <ul>
      <li>研究 Agent：收集资料、竞品分析、用户反馈归类</li>
      <li>产品 Agent：生成需求结构、功能清单和优先级建议</li>
      <li>原型 Agent：生成页面代码、交互 Demo 和数据样例</li>
      <li>评审 Agent：检查逻辑漏洞、边界条件和上线风险</li>
    </ul>
  </article>
</section>

## 一个产品经理的实操闭环

<section class="industry-process">
  <div>
    <span>Step 1</span>
    <h3>输入业务问题</h3>
    <p>用自然语言描述业务背景、目标用户、当前痛点和希望达成的结果。</p>
  </div>
  <div>
    <span>Step 2</span>
    <h3>生成需求结构</h3>
    <p>让 AI 输出用户故事、功能边界、非目标范围、成功指标和验收标准。</p>
  </div>
  <div>
    <span>Step 3</span>
    <h3>快速做出原型</h3>
    <p>用 Vibe Coding 生成一个可运行页面或交互 Demo，让讨论不再只停留在文字。</p>
  </div>
  <div>
    <span>Step 4</span>
    <h3>Agent 评审迭代</h3>
    <p>让不同 Agent 从用户、开发、运营、老板视角审查方案，并给出修改建议。</p>
  </div>
</section>

## 课堂练习

你将选择一个真实产品问题，例如“提升新用户激活率”“优化客服工单分流”“设计一个 AI 简历分析工具”，并完成：

- 一份结构化需求说明
- 一份用户故事和验收标准
- 一个可演示的网页原型
- 一轮多 Agent 评审记录
- 一份面向老板或客户的方案汇报
