# 学生的 Vibe Coding 工作流

<section class="industry-hero">
  <p>Industry Scenario 02</p>
  <h2>从完成作业，到独立做出可演示项目。</h2>
  <span>作业 / 竞赛 / 科研展示 / 创业原型</span>
</section>

学生最需要的不是“多学一个工具”，而是把课堂知识、研究资料、代码实验和展示表达串成一个完整项目。Vibe Coding 能让学生更早进入“独立完成作品”的状态：你提出问题，AI 帮你拆解路径、生成代码、解释错误、整理材料，并把成果变成可以展示的 Demo。

## 典型任务

<section class="industry-tasks">
  <article><h3>课程作业</h3><p>理解题目要求，生成解题思路、代码示例、实验报告和展示材料。</p></article>
  <article><h3>竞赛项目</h3><p>快速搭建原型，完成数据处理、页面展示、模型调用和答辩材料。</p></article>
  <article><h3>科研展示</h3><p>整理文献、提炼观点、制作图表、生成摘要和演示页面。</p></article>
</section>

## 三种工作流对比

<section class="workflow-grid">
  <article><strong>01</strong><h3>古法工作流程</h3><p>靠搜索、教程和同学帮助推进任务，遇到代码报错或环境问题就容易卡住。</p><ul><li>资料散落在网页、论文和聊天记录里</li><li>代码能复制但不一定理解</li><li>项目结果难以整理成完整展示</li><li>遇到问题常常不知道该问什么</li></ul></article>
  <article><strong>02</strong><h3>对话式 AI 工作流程</h3><p>把 AI 当成学习助教和项目搭档，通过对话完成理解、生成、解释和修改。</p><ul><li>让 AI 解释题目和知识点</li><li>生成代码骨架和实验步骤</li><li>解释报错并给出修复建议</li><li>把过程整理成报告和答辩稿</li></ul></article>
  <article><strong>03</strong><h3>Agents 工作流</h3><p>把资料、代码、检查和展示拆给不同 Agent 协作，形成完整项目闭环。</p><ul><li>研究 Agent 整理背景资料</li><li>开发 Agent 生成 Demo 和脚本</li><li>评审 Agent 检查逻辑和引用</li><li>演示 Agent 生成汇报结构</li></ul></article>
</section>

## 一个学生的实操闭环

<section class="industry-process">
  <div><span>Step 1</span><h3>明确题目</h3><p>把作业、竞赛或研究问题转化成清晰目标、输入、输出和限制条件。</p></div>
  <div><span>Step 2</span><h3>生成方案</h3><p>让 AI 拆解任务路径，列出资料、代码、实验和展示所需内容。</p></div>
  <div><span>Step 3</span><h3>跑通 Demo</h3><p>生成可运行代码，调试环境问题，并保存关键运行结果。</p></div>
  <div><span>Step 4</span><h3>完成展示</h3><p>整理报告、PPT、演示页和答辩问题，让成果可以被别人看懂。</p></div>
</section>

## 课堂练习

你将选择一个课程或竞赛主题，例如“校园活动报名系统”“AI 学习助手”“论文摘要工具”，并完成：

- 一份项目 brief
- 一段可运行 Python 或网页代码
- 一份项目说明文档
- 一页可演示作品页
- 一份答辩问题清单

## 学完后你能做到

<section class="industry-outcomes">
  <div><h3>独立完成项目</h3><p>从题目理解到代码运行，再到成果展示，形成完整交付能力。</p></div>
  <div><h3>更会向 AI 提问</h3><p>把模糊问题拆成 AI 能执行的任务，减少无效搜索和反复试错。</p></div>
  <div><h3>更快展示成果</h3><p>把代码、报告和演示页组合成可被老师、队友或评委理解的作品。</p></div>
</section>
